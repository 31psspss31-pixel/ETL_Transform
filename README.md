# Практика в Атомэнергопроект

## Цель работы
Создать программное решение, которое формирует историю изменений атрибутов объектов на основе предоставленных данных в формате CSV, при помощи ETL процесса, а именно Transform этапа. Результатом работы является файл `history_summary.csv`, содержащий полную временную развертку состояний объектов с их атрибутами.

## Задачи
1. Разработать структуру классов для хранения информации об объектах (`ObjItem`) и их атрибутах (`AttributeItem`).
2. Реализовать чтение данных из CSV-файлов (`processed_obj.csv`, `processed_attr.csv`).
3. Обеспечить корректную обработку временных меток (дата создания и завершения атрибутов/объектов).
4. Сформировать таблицу, в которой для каждого объекта отражены все изменения его атрибутов с указанием периодов актуальности.
5. Сохранить результат в новый CSV-файл (`history_summary.csv`).

## ETL процесс

ETL (Extract, Transform, Load) — это не единый алгоритм, а процесс, состоящий из последовательности шагов для обработки данных.

### Основные этапы ETL-алгоритма:

1. **Extract (Извлечение)** - Получение данных из различных источников:
   - Реляционные БД (MySQL, PostgreSQL)
   - NoSQL-хранилища (MongoDB, Cassandra)
   - Файлы (CSV, JSON, XML)
   - API веб-сервисов
   - Потоковые данные (Kafka, RabbitMQ)

2. **Transform (Трансформация)**
   - Очистка данных: удаление дубликатов, исправление ошибок, заполнение пропусков
   - Преобразование: нормализация/денормализация, агрегация (суммы, средние), сортировка и фильтрация
   - Обогащение: добавление вычисляемых полей, объединение данных из разных источников

3. **Load (Загрузка)** - Запись преобразованных данных в целевую систему:
   - Data Warehouse (Snowflake, Redshift)
   - Data Lake (Hadoop, S3)
   - Операционные БД
